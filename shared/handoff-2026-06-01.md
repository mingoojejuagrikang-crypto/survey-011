# 핸드오프 — 2026-06-01 (v0.11.1)

## 이번 작업 요약

iPhone Safari 실기기 테스트(2026-06-01, 마이크 "OpenDots ONE by Shokz")에서 보고된 3가지를 다룸:

1. 데이터탭 음성 클립 재생 버튼 미동작
2. 로그 ZIP에 음성 클립 누락
3. 소음(비 오는 비닐하우스) 환경 인식률 저하

### 진단 (오늘 로그 `growth-log_2026-06-01_*.zip` 분석)
- value 45건 정상, **clip 에러 로그 0건·ZIP에 `clips/` 폴더 없음** → 클립이 IDB에 아예 저장 안 됨.
- 코드 분석: `saveAudioClip`이 **Blob을 IDB에 직접 저장**(`db.put('audioClips', blob, key)`)했는데, iOS Safari의 Blob-in-IndexedDB 실패를 `useVoiceSession.ts`의 **빈 `catch{}`가 로그 없이 삼킴**. v0.10·v0.11이 두 번 고치고도 실패한 이유 = 확인 없는 추측 수정.
- **#1·#2는 같은 단일 근본 원인**(IDB 저장 실패)을 공유. `exportLog.ts`는 IDB를 직접 읽으므로 저장만 고치면 둘 다 해결.

### 적용한 수정
- **db.ts**: 클립을 `{buf:ArrayBuffer, type}`로 저장 (iOS 우회) + 구형 Blob 하위호환.
- **audioRecorder.ts**: `recorder.start(250)` timeslice, `stopClip` 2초 타임아웃(hang 방지), `startClip` try/catch + 진단 로그(`clip_started`/`clip_no_stream`/`clip_start_failed`/`clip_stop_timeout`), `stopTimer` 전 경로 정리. getUserMedia에 `noiseSuppression`/`echoCancellation` on, `autoGainControl` off(소음 증폭 방지).
- **useVoiceSession.ts**: 빈 catch에 `clip_save_failed` 로깅 + breadcrumb(`clip_stop_await`/`clip_stop_resolved:size`/`clip_saved`), 터치 명령(`restart`/`jump`/`touch_commit`)·`pause`/`resume` 로깅, pause 시 pending save flush.
- **logger.ts**: `'clip'` 타입 추가 + 세션 컨텍스트 자동 첨부(recorder 내부 로그도 세션 필터 ZIP에 포함).

QA: Codex adversarial review 6건 중 HIGH 2건(timeslice·sessionId)·MEDIUM 1건(pause flush) 반영.

## ⚠️ 중요: 이것은 "수정 완료"가 아니라 "진단 빌드"
테스트가 이 버그를 못 보므로, **다음 iPhone 실기기 재테스트 1회로만 확정**된다. 재테스트 후 새 ZIP의 events.json을 다음 순서로 읽을 것:
- `clip_started` 없음 → stream/init 타이밍 문제
- `clip_stop_await`는 있는데 `clip_stop_resolved` 없음 → 가설(a): 마이크 점유로 stopClip hang
- `clip_stop_resolved:>200`인데 IDB/ZIP에 클립 없음 → 가설(b) 잔존 (다른 저장 실패)
- `clips/` 폴더에 mp4 존재 → **#1·#2 해결 확인**

## 후속 백로그 (Codex 잔여 지적, 진단 데이터 확인 후 판단)
- **HIGH 2**: clip 저장 실패 시 이미 persist된 세션 row의 `audioClips`에 stale key가 남음 → "재생버튼 보이나 클립 없음" 가능. (b) 해결되면 발생 빈도 낮음.
- **MEDIUM 4**: `stopClip()` 재진입 시 첫 Promise 영구 pending (현재 hook은 중복 호출 안 함).
- **MEDIUM 5**: 재시도 경로 즉시 stop→start가 iOS에서 recorder 공백 유발 가능 → `clip_start_failed` 로그로 관찰.
- **노이즈 #3-C**: 입력 장치 선택 UI(붐마이크/유선 권장). 범위 커서 별도 작업.
- **부차 UX**: 오늘 로그에 "수정" 14회 연속 — 정정 모드 진입/진행이 막혔을 가능성. 재현·조사 필요.
